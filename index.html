HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The target attribute specifies where to open the linked document.
  • The p element divides content into easy to read segments.
  • The id attribute is a way to identify elements and select them.
  • The ul and li elements are used to create lists.
    • Metadata contains information about accessibility, search engine information, and performance.
  • Elements tell the browser what to display.
  • Attributes define how things will be shown. They're always placed in the opening tag of an element.
  • Tags mark the beginning and end of an element.

CSS

  • CSS is a programming language used by developers to define how webpage content is being presented to users.
  • CSS defines the color, aesthetic, visual layout, and organization.
  • There are three ways to style a webpage using CSS: inline, internal style sheet, and external style sheet.
  • We can link an external style sheet using an link element.
  • A selector defines the element or attribute to which the rules/declarations will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • Every CSS element is essentially a box, and each box consists of margins, padding, and borders.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • We combine two or more elements into a single CSS rule with a comma, allowing the declarations to affect both the selectors.
  • DRY stands for Do Not Repeat Yourself and is a principle that encourages developers to reuse/share pieces of code in order to reduce the number of lines of code that need to be written/maintained.
  • text-align is a rule assignment for positioning
  • display-block assigns each element to start on a new line.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project.
  • To access Git we use the command line.
  • Git is great for tracking changes locally on your own computer, but to collaborate with others, you need to use GitHub
  • Git commands always start with 'git' and are used to save and track versions of your code.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it
  • ssh -T git@github.com checks the connection of the SSH key to your GitHub

JavaScript

  • JavaScript is a high performance/dynamic programming language that allows users to interact with websites.
  • A variable is a named container that allows us to store data in our code.
  • A string is a sequence of text enclosed in quotation marks.
  • A Boolean is a True/False value.
  • Control flow is the order in which a computer executes code in a script.
  • An array allows you to store multiple vaules in a single reference.
  • We use an index to locate/access individual data inside the array.
  • = is an assignment value.
  • === is strict equality, which performs a test to see if the two values are equal and of the same data type.
  • A function is a way of packing functional code that you wish to reuse.
  • To declare a variable in Javascript, we can use const, var, or let.
  • Conditionals are code structures used to test an expression. They return true or not.
  • We use the console.log method to output a message to the web console by adding an argument.
  • The word console refers to the test environment that developers use to test and debug code.
  • .log() is a method that will output whatever is inside the parentheses.
  • A method is a set of instructions that tell the computer how to execute.
  • To link Javascript to HTML file, we use a script element in the footer.
  • To see the console.log in action we use the Chrome DevTools.
  • We use 'command+option+I to open the developer tools.
  • An if statement is a way of introducing decision making into code.
  • A for loop uses the predictable pattern of indices to performa a task on all the items in an array by allowing a single code block to be executed over and over.
  • We can find an arrays length by using a build in property named .length
  • Let allows us to reassign the value of the variable over time.
